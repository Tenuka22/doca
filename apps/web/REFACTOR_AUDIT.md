## Section B ‚Äî Inconsistencies

### B1. Query pattern: `queryOptions` vs inline `useQuery` (P1-Critical)
- **Uses `queryOptions()` (canonical):** `doctor/profile.tsx`, `doctor-profile-card.tsx`, `use-doctor-files.ts`
- **Uses inline `{ queryKey, queryFn }` (non-canonical):** All other route files

### B2. Loader pattern: `ensureQueryData` vs `prefetchQuery` vs `fetchQuery` (P2-High)
- **Uses `prefetchQuery` (non-canonical):** `admin/index.tsx`, `doctor/index.tsx`, `doctor/sessions/index.tsx`, `doctor/availability.tsx`, `doctor/plans.tsx`, `doctor/profile.tsx`
- **Uses `fetchQuery` (non-canonical):** All admin list route loaders
- **Uses `ensureQueryData` (canonical):** None

### B3. Loader vs no-loader (P3-Medium)
- **Routes WITH loader prefetch:** `/admin/`, `/admin/doc-requests/`, `/admin/doctors/`, `/admin/doctors/$doctorId`, `/admin/guardians/`, `/admin/patients/`, `/admin/plans/`, `/admin/sessions/`, `/admin/credits/`, `/doctor/`, `/doctor/sessions/`, `/doctor/availability/`, `/doctor/plans/`, `/doctor/profile/`
- **Routes WITHOUT loader prefetch:** `/admin/session`, `/doctor/credits`, `/doctor/sessions/$sessionId`

The non-prefetched routes are appropriate (test session, credits page, session detail) ‚Äî no change needed.

### B4. Mutation error handling (P2-High)
- `onError` in `useMutation` options: `doctor/credits.tsx`, `doctor/availability.tsx`, `doctor/plans.tsx`, `doctor-profile-card.tsx`
- try/catch around `.mutateAsync()`: `doctor-files-panel.tsx`
- `onSuccess` with no error handling: `admin/doc-requests/index.tsx`

**Proposed source of truth:** Use `onError` consistently in `useMutation` options via `notify.error()`.

### B5. Post-mutation invalidation (P2-High)
